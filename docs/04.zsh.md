# ZSH Configuration Documentation

## Overview

This document explains how ZSH is organized in this dotfiles setup.

Configuration is split by responsibility:

- `env.d`: environment variables loaded very early (`.zshenv`)
- `conf.d`: interactive shell behavior loaded in `.zshrc`
- `zsh-abbr`: persistent abbreviations file

## Directory Structure

### `.config/zsh/` - Main ZSH Modules

- **Purpose**: Home of `.zshenv`, `.zshrc`, and modular scripts
- **Location**: `~/.config/zsh`
- **Key Modules**:
  - `env.d/` for exported environment variables
  - `conf.d/` for interactive runtime configuration
  - `completions/` for custom completion definitions

### `.config/zsh-abbr/` - Abbreviations Storage

- **Purpose**: Storage for `zsh-abbr` definitions
- **Location**: `~/.config/zsh-abbr/user-abbreviations`

## Configuration Loading Mechanism

### `.zshenv` loads `env.d`

`dot_zshenv.tmpl` loads all readable `*.zsh` and `*.sh` files from:

- `${ZDOTDIR:-$HOME/.config/zsh}/env.d`

Lexical filename order controls load order, so numeric prefixes matter.

### `.zshrc` loads `conf.d`

`dot_zshrc.tmpl` loads interactive config from:

- `${ZDOTDIR:-$HOME/.config/zsh}/conf.d`

It includes both top-level and one-level nested files:

- `conf.d/*.{zsh,sh}`
- `conf.d/*/*.{zsh,sh}`

## Recommended Naming

Use numeric prefixes for deterministic ordering:

- `000-xxx` to `099-xxx` in `env.d`
- `001-xxx` to `090-xxx` in `conf.d`

Current `env.d` example:

```text
env.d/
├── 000-xdg.sh.tmpl
├── 010-homebrew.sh.tmpl
├── 020-common.sh.tmpl
├── private_030-secrets.sh.tmpl
├── encrypted_private_030-secrets-age.sh
└── 040-path.sh.tmpl
```

## Aliases and Abbreviations

### Aliases

Aliases are defined in `conf.d` modules (for example `060-common-aliases.sh.tmpl`) and any third-party scripts under `conf.d/third-party/`.

### zsh-abbr

Abbreviations expand inline as you type and are stored in:

- `$ABBR_USER_ABBREVIATIONS_FILE`

Useful commands:

```bash
abbr list
abbr import-aliases
```

## Troubleshooting

### Verify Files Exist

```bash
ls -la ~/.config/zsh/env.d
ls -la ~/.config/zsh/conf.d
```

### Trace Startup Loading

```bash
ZSH_DEBUG=1 zsh -ic exit
```

### Deep Startup Trace

```bash
zsh -xilc exit 2>&1 | rg "source|Loading config file"
```

### Syntax Check

```bash
zsh -n ~/.config/zsh/.zshenv
zsh -n ~/.config/zsh/.zshrc
```

## Maintenance

1. Keep environment exports in `env.d`.
2. Keep interactive behavior in `conf.d`.
3. Keep secrets in `private_*` / encrypted files.
4. Preserve numeric prefixes to avoid order regressions.
